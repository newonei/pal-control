# 安全策略

Pal Control 仍处于开发服阶段。请勿把管理 API、Palworld 官方 REST、PalDefender、RCON 或 Native Named Pipe 直接暴露到公网。

## 支持范围

| 版本 | 安全更新 |
| --- | --- |
| `main` 当前版本 | 是 |
| 历史提交、第三方打包和自行修改版本 | 否 |

项目尚未发布稳定版；安全修复会先进入 `main`，不承诺为旧提交单独回移。

## 私下报告漏洞

请使用 GitHub 仓库的 **Security → Report a vulnerability** 创建私密 Security Advisory。不要为认证绕过、任意命令执行、存档破坏、重复入账、凭据泄露或玩家隐私问题创建公开 Issue。

报告建议包含：

- 受影响的提交、Palworld/UE4SS/PalDefender 版本组合；
- 可复现的最小步骤、预期结果与实际结果；
- 影响范围，以及是否可能造成游戏写入、经济重复结算或数据泄露；
- 已删除 Token、Cookie、密码、玩家标识、真实存档和本机路径的日志或测试用例。

请勿上传真实 `.sav`、SQLite 数据库、崩溃转储、服务器配置或凭据。维护者会在私密渠道确认影响、协调整改和公开时间；修复发布前请不要公开利用细节。

## 不属于漏洞的情况

- 已在 README/TODO 中明确标记为 fail-closed 或尚未支持的能力；
- 仅能在已经拥有服务器管理员或本机管理员权限后修改本机文件；
- 未经授权对第三方 Palworld 服务器进行扫描或测试。

即使不确定问题是否属于安全漏洞，也请优先使用私密报告渠道。
