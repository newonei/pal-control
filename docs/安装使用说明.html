幻兽商域

Windows 安装与使用说明 · 面向无需开发环境的服务器管理员

先看这里

安装包已经包含管理网页、玩家网页和 .NET 运行环境。使用安装包时，不需要安装 Node.js、npm、Visual Studio 或 .NET SDK。

幻兽商域是 Palworld Dedicated Server 的本机管理工具，不包含 Palworld 服务端本体。开始前，请先在同一台 Windows 电脑上安装好 Palworld Dedicated Server。

一、安装前准备

项目要求
操作系统64 位 Windows 10、Windows 11 或 Windows Server
Palworld 服务端已经安装，并能正常启动进入世界
白名单资源目录基础管理台不需要；启用玩家门户、商城或资源兑换前，管理员必须自行提供经授权的 <安装目录>\Resources\palworld-resource-catalog.json（源码运行时对应 services/control-api/Resources）。公开安装包因再分发授权未闭环而不包含它
权限幻兽商域采用当前用户安装，通常不需要管理员权限
网络管理台默认只在本机 127.0.0.1:5180 打开
先备份存档。在修改服务器配置或安装任何 MOD 之前，先完整备份 Pal\Saved\SaveGames。

二、安装幻兽商域

运行安装程序双击当前发布版本的 幻兽商域-Setup-<版本>.exe。选择安装位置，普通用户保持默认即可。
处理 Windows 提示当前安装包没有商业代码签名证书。若 SmartScreen 显示“Windows 已保护你的电脑”，先确认文件来自可信渠道并核对随包提供的 SHA-256，再选择“更多信息 → 仍要运行”。
创建快捷方式建议勾选“创建桌面快捷方式”。开始菜单会同时提供启动、停止、配置和本说明。
打开配置窗口安装结束时保持“立即配置”勾选；以后也可从开始菜单打开“配置幻兽商域”。

三、启用 Palworld 官方 REST

幻兽商域通过 Palworld 官方 REST 读取服务器状态和在线玩家。请先停止 Palworld 服务端，再编辑：

PalServer\Pal\Saved\Config\WindowsServer\PalWorldSettings.ini

确认 OptionSettings=(...) 中包含以下设置；不要创建第二个 OptionSettings：

  • AdminPassword="一个强密码"
  • RESTAPIEnabled=True
  • RESTAPIPort=8212

保存后重新启动 Palworld 服务端。管理员密码不要发给玩家，也不要放进截图或群聊。

四、首次配置

选择 PalServer 目录选择包含 PalServer.exe 的文件夹。通常是 Steam 库中的 Palworld Dedicated Server 安装目录。
填写 REST 端口没有自行修改时保持 8212。
填写管理员密码这里必须与 PalWorldSettings.ini 的 AdminPassword 完全一致。
保存配置与管理员凭据配置保存在当前用户的安装目录中。首次保存会生成管理 API Key 与 TOTP 密钥且只显示一次，请立即放入密码管理器；修改旧配置时，工具会先生成带时间戳的备份，只覆盖界面中的 PalServer 目录、REST 地址、账号和密码，已启用的 Native、PalDefender、玩家门户及未知扩展字段会保留。

五、启动和使用

  1. 先启动 Palworld Dedicated Server，并等待世界加载完成。
  2. 双击桌面的“幻兽商域”，或从开始菜单打开“启动幻兽商域”。
  3. 程序会在后台启动，并自动打开 http://127.0.0.1:5180/。
  4. 页面出现管理员登录门禁后，粘贴首次配置时只显示一次的 API Key。不要把 API Key 写入网址、截图或浏览器同步笔记。
  5. 登录后先查看仪表盘。Control API 应显示在线；官方 REST 连接成功后，服务器信息和在线玩家会出现。
  6. 使用结束后，从开始菜单打开“停止幻兽商域”。

关闭浏览器不会停止后台服务。重复点击启动快捷方式不会重复启动，只会重新打开管理台。

六、功能开放顺序

阶段可用功能建议
首次安装管理台、健康检查、本地队列先确认页面能正常打开
官方 REST 已连接服务器状态、在线玩家、实时地图、保存请求先使用只读功能验证配置
PalDefender 与玩家门户已配置游戏内验证码登录、商城发货、奖励、公会和封禁必须限制为本机访问、使用最小权限 Token；验证码还要求本机受限 RCON 精确提供 send:3
公网 Steam 门户Steam OpenID + 当前周角色验证码双层登录必须使用官方 HTTPS OP、精确 realm/return_to、安全 Cookie 和审核后的 Caddy 反向代理；不能把管理 API 暴露到公网
精确版本 Native MOD背包/帕鲁精细工具、Native 通知、白名单资源兑换资源兑换还必须取得稳定 inventory.consume 能力并通过保存、停服、重启、重新登录验收
受限 RCON玩家验证码私信；可选开发诊断和经批准的人工应急启用玩家门户时它是必需的本机消息通道；正式资源兑换不会使用 RCON 模拟扣物，也不要对公网开放端口
玩家门户、PalDefender、RCON 和 Native 写能力默认关闭。这是安全设计，不是安装失败。当前资源兑换在 Native 持久化证据未通过时会保持只读；不要为了“全部变绿”而直接开放端口、伪造能力或跳过版本检查。

公网 Steam 登录的完整配置不是安装向导默认开启项。请按 GitHub 仓库中的 玩家门户部署手册 配置域名、TLS、Caddy、Steam OpenID、PalDefender REST、经授权的白名单资源目录和仅回环可达的 RCON 密码文件，再执行验收清单。发布包只附带本说明，不包含仓库的完整运行手册。OpenID 只证明 Steam 身份；游戏内验证码成功并确认本周 worldId + PlayerUID 后才会建立正式会话。

资源经济前置条件：公开安装包不包含 palworld-resource-catalog.json。缺少管理员自行提供的经授权目录时，基础管理页仍可使用，但玩家绑定、商城初始化和白名单资源兑换会保持不可用；不要从不明来源下载或重新分发游戏数据快照。

七、常见问题

启动后没有自动打开网页

手动打开 http://127.0.0.1:5180/。若仍打不开，先运行“停止幻兽商域”，再重新启动；同时检查其他程序是否占用了 5180 端口。

网页能打开，但服务器显示离线

确认 Palworld 服务端正在运行，RESTAPIEnabled=True，REST 端口和管理员密码与配置窗口完全一致。修改后需要重启 Palworld 服务端。

看到“待连接”“安全只读”或灰色按钮

对应的外部组件尚未启用或版本门禁没有通过。基础安装默认如此。先完成官方 REST 只读联调，再按照项目运行手册逐项启用进阶组件。

怎样查看错误日志

打开安装目录中的 logs 文件夹。普通运行数据在 data，配置文件为 appsettings.Local.json。向维护人员求助时可提供日志，但应先检查并遮盖服务器路径、Token 和密码。

升级会不会删除数据

覆盖安装前仍建议备份安装目录中的 appsettings.Local.json 和 data。卸载程序不会主动删除运行后生成的配置和业务数据；确认不再需要后可手工删除残留目录。

八、安全要求

  • 不要在路由器、云安全组、端口映射或内网穿透中开放 5180、8212、17993、25575。
  • 管理台只应在服务器本机使用。需要远程管理时，优先使用系统远程桌面或受控 VPN。
  • 公网只能通过经过审核的 HTTPS 反向代理开放玩家门户和 /api/v1/player/*，不能开放管理 API。
  • 任何显示为 uncertain 的发货或写操作都要先人工核对，不要重复点击。
  • Native MOD 当前目标是 Palworld v1.0.0.100427 与指定 UE4SS 提交。公开发布包不包含 Native DLL；必须按仓库锁定版本单独获取依赖并自行构建，版本不一致时不要加载。

九、卸载

  1. 从开始菜单打开“停止幻兽商域”。
  2. 在 Windows“设置 → 应用 → 已安装的应用”中找到“幻兽商域”并卸载。
  3. 如需彻底清理，先确认不再需要账本、审计和配置备份，再手工删除残留的安装目录。
