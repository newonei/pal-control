# 玩家门户上线验收清单

## A. 构建物

- [ ] `apps/player-web/dist/index.html` 存在，构建来自已审查提交和锁定依赖。
- [ ] `dist` 中没有 `.env`、私钥、服务器配置、真实玩家数据或运营控制台页面。
- [ ] 若保留 source map，已确认其中没有秘密和不应公开的源码；否则从生产构建移除。
- [ ] 前端所有 API 请求使用同源相对路径 `/api/v1/player/...`，没有硬编码 `5180`、管理员密码或内部 token。

## B. Caddy 配置

- [ ] 三个 `PLAYER_PORTAL_*` 变量均由服务环境提供，仓库中没有真实域名或秘密。
- [ ] `caddy fmt --diff` 没有意外格式变化。
- [ ] `caddy validate --config <path> --adapter caddyfile` 成功。
- [ ] `caddy adapt --pretty` 中只有 `127.0.0.1:5180` 一个后端。
- [ ] 唯一代理匹配器为 `/api/v1/player/*`，其他 `/api` 由 `404` handler 终结。
- [ ] Caddy Admin API 只监听 `127.0.0.1:2019`。
- [ ] Caddy 服务账户只能读取静态文件、写日志和 Caddy data，不能读取 Palworld 凭据。
- [ ] 任意点文件路径返回 `404`；超过 16 KiB 的玩家 API 请求体在 Caddy 边界返回 `413`。

## C. DNS、TLS 与端口

- [ ] 公共 `A` 记录正确；仅在 IPv6 完整可用时保留 `AAAA`。
- [ ] HTTP 自动跳转 HTTPS，证书链有效，域名匹配且续期状态正常。
- [ ] 仅 `80/TCP`、`443/TCP`（以及选用的 `443/UDP`）为门户开放。
- [ ] 外网无法连接 `5174`、`5180`、`8212`、`17993`、`25575`、`2019`。
- [ ] Palworld 的 `8211/UDP`、`27015/UDP` 规则没有被误改为 TCP，也没有把管理端口一并映射。

## D. 路由黑盒测试

在真正的外部网络执行，不能只在服务器本机测试。PowerShell 示例中的域名必须替换：

```powershell
$Domain = "<player-domain>"

curl.exe -sS -D - -o NUL "http://$Domain/"
curl.exe -sS -D - -o NUL "https://$Domain/"
curl.exe -sS -D - "https://$Domain/api/v1/player/me"
curl.exe -sS -D - "https://$Domain/api/v1/extraction/overview"
curl.exe -sS -D - "https://$Domain/api/v1/servers/local/info"
curl.exe -sS -D - "https://$Domain/api"
curl.exe -sS -D - "https://$Domain/api%2Fv1%2Fservers%2Flocal%2Finfo"
```

预期结果：

- [ ] HTTP 返回到同域 HTTPS 的永久重定向。
- [ ] `/` 返回门户 HTML；浏览器刷新一个 SPA 子路由仍返回门户，而不是 Caddy 404。
- [ ] 未登录访问实际存在的玩家端点得到受控 `401/403`；登录后只能看到当前账户数据。
- [ ] extraction、servers、`/api` 和编码后的非玩家 API 均为 `404`，且请求没有出现在 Control API 的管理路由日志中。
- [ ] `/api/v1/player`（无尾随 `/`）为 `404`，符合严格白名单。
- [ ] 大小写、双斜线、点段、百分号编码和超长路径不会绕过 API 边界。

## E. 安全响应头和浏览器行为

- [ ] HTTPS 响应包含 HSTS、CSP、`nosniff`、拒绝 frame、Referrer Policy 和 Permissions Policy。
- [ ] 响应不暴露 `Server` 版本。
- [ ] CSP 控制台没有被忽略的真实错误；没有为修错误加入 `*` 或 `'unsafe-eval'`。
- [ ] 登录 Cookie 为 `Secure`、`HttpOnly`，并设置与流程匹配的 `SameSite`。
- [ ] 所有 Cookie 写操作通过 CSRF 和 Origin 检查；CORS 不是唯一防护。
- [ ] 登出后会话立即失效，旧 Cookie 不能继续访问玩家数据。
- [ ] 认证路径与 `/me` 路径超过固定窗口阈值时返回 `429` 和整数秒 `Retry-After`；并发超过硬上限时立即拒绝而不排队。

## F. 业务授权与幂等

- [ ] 修改请求正文或 URL 中的玩家/订单 ID 不能访问另一账户。
- [ ] 同一个购买幂等键重放返回同一结果，不重复扣款或发货。
- [ ] 同一个键配不同请求返回冲突；超时后客户端先查订单状态，不盲目换键重试。
- [ ] 玩家 API 不能调用调账、人工对账、换档、RCON、公告或服务器配置接口。
- [ ] 失效、封禁或退出登录的会话不能继续创建写操作。

## G. 运维与回滚

- [ ] 访问日志可写、JSON 可解析、敏感请求头保持脱敏，日志会滚动且磁盘告警有效。
- [ ] Caddy data/证书状态位于持久目录并有受控恢复方案。
- [ ] 已保存上一个静态版本，切回并 reload 的演练成功。
- [ ] Control API 不可用时门户显示受控故障，不泄露内部地址或异常堆栈。
- [ ] 触发存储、状态冲突和未知异常时，玩家响应不包含本机绝对路径、SQLite 文件名或堆栈。
- [ ] 有证书续期、`5xx`、异常登录、限流触发和磁盘空间告警。
- [ ] 上线批准记录包含前端版本、Caddyfile 哈希、Control API 版本、验收时间和执行人。
