# 玩家使用教程

本教程描述方案 A“周世界资源经济服”的当前玩家流程。服务器管理员必须先完成玩家门户、身份绑定、PalDefender、授权资源目录和 Economy Safety Gate 配置；若页面显示维护、版本不匹配或能力未就绪，请按提示等待管理员处理，不要反复提交。

> 当前 dev39-ro 是 `quarantined` 只读 Native 源码/制品候选，不发布扣物或其他写能力。两个独立锁定构建已得到 893,440 字节、SHA-256 `c2dab9f9bfd3c47ac1a244139fb96ce1de6f598c4bce438ebddde96185063b34` 的字节一致 DLL，但尚未实服加载或运行固定探针套件。9 项非玩家成功、3 项无人在线拒绝、0 项意外失败属于已 superseded/quarantined 的 dev38-ro 历史证据，不能转记给 dev39-ro；旧 dev36 experimental 与缺陷 dev37-ro 也仅是历史。只有 dev39-ro 固定套件、受控在线玩家 3 项探针、PalDefender 组合与独立复核通过、另行评审写候选并完成真实“扣物 -> 保存 -> 停服 -> 重启 -> 重登”验收后，Production 资源兑换才可能开放；官方保存和优雅关服本身不等于该验收。本文是受控开发服使用说明，不代表项目已经可以直接对公网开放。

## 1. 登录并绑定本周角色

1. 公网 Steam 模式会跳转到 Steam 官方 OpenID 页面。幻兽商域不会要求输入 Steam 或 Palworld 密码。
2. Steam 验证成功后只有短时待绑定身份，尚不能查看钱包或交易。
3. 保持本周 Palworld 角色在线，网页请求验证码；服务器通过本机受限通道向该角色私发 8 位一次性验证码。
4. 在网页提交验证码。服务端实时确认本周 `worldId + PlayerUID` 后才建立正式会话。
5. 可信好友服可能显式使用“仅游戏验证码”模式；页面会标出这是 fallback。不要把验证码、Cookie 或会话链接发给他人。

登录后首页按真实账户状态显示 5 步引导：登录绑定、购买补给、进入资源兑换点、选择并确认出售、查看资金账本。步骤完成状态来自服务端订单、位置、报价、兑换和账本，不需要玩家手工打勾。

## 2. 领取新玩家活动

若当前周世界发布了新玩家活动，首页会显示版本、奖励与是否已领取。点击领取才会入账；同一活动版本只能领取一次，刷新或重放不会重复发奖。没有已发布活动时页面不会虚构默认奖励；开发兼容配置中的初始余额不代表正式服规则。

## 3. 查看任务

“本周可靠任务”显示日/周任务、当前进度、奖励、任务积分以及固定的规则/内容证据。内置内容候选为 3 个日任务和 3 个周任务，只累计服务端已经证明的经济事件，例如成功资源兑换、指定资源/价值结算、商城成功送达和指定货币消费。同一事件重放不会重复推进，同一奖励只入账一次。

击杀、采集、死亡、PvP 或进入热点目前不属于可靠任务事件；若页面没有这些任务，这是预期的安全边界。

## 4. 参加本周小队协作

“团队协作”是当前服务器、当前周世界内的经济目标，不等同于 PalDefender 公会，也不能跨服务器组队、转币或交易。

1. 没有小队时，可输入 2–32 字符队名创建；同一周世界中的规范化队名不能重复。也可粘贴队长发来的邀请 token 加入。token 只用于本次提交，不会写入浏览器本地存储。
2. 队长生成或轮换邀请后，明文只在首次成功响应显示一次；响应丢失后的安全重放不会再次显示。邀请可有有效期和使用次数上限，怀疑泄露时应立即轮换，旧 token 随即失效。
3. 页面显示四个冻结目标：出售白名单资源件数、累计资源价值、可靠任务积分和成功送达订单。达标只记录里程碑，**不会额外发放商域币、战备券或物品**。
4. 贡献只统计你加入之后、离队之前的权威终态。加入前历史不会倒灌，离队后的新贡献不再计算，换队不会把旧贡献搬到新队。采集、击杀、死亡、PvP 或浏览器上报不属于团队事实。
5. “团队累计”是队内汇总，“我的贡献”只属于当前登录者。三张公开榜分别按资源价值、可靠任务积分和成功送达排序；它们只显示队名、合格成员数和聚合值，不显示成员账号、PlayerUID、平台身份或显示名。真实身份封禁或赛季人工排除会立即从当前成员门槛、贡献和榜单剔除；解除后按同一权威事实确定恢复，已达到的无奖励目标里程碑不会倒退。
6. 队长可使用“成员 XXXX”的临时不透明标签转让队长，不会看到平台账号。旧标签不能把队长转给已封禁/排除成员；队长被封禁后旧邀请也不能继续加入，需管理员审计解封/取消排除后再转让或解散。队长必须先转让或输入完整队名解散，不能直接离队；普通成员可确认离队。
7. “等待权威投影”表示首次统计尚未完成，页面不会用假 0 代替未知。“使用上次安全快照”表示新一轮重算失败或已过期，应保留当前显示并等待管理员排查，不要重复购买、兑换或退出重进来推动它。

团队目标在创建时按模板冻结，运营周中调整默认值不会改写既有队伍。真实贡献争议应由管理员核对结算、订单、任务积分和成员时间窗，不以客户端截图人工补分。

## 5. 购买战备商品

1. 打开“战备商城”，按分类、标签或关键字筛选。推荐位、分类和标签由当前内容版本明确给出；商品卡片同时显示仓库内置图标、五档稀有度和用途说明，不会从外部网址加载图标。
2. 区分两种数量：
   - “个人本周剩余”是你的周限购余量；
   - “全服剩余”只在商品配置了服务器总库存时显示，所有玩家共享。未配置全服库存时页面不会显示一个虚假库存数。
3. 核对商品、货币、当前价格和发放摘要后购买。目录每天按业务日生成版本化价格；默认日价为基础价的 90%–110%，当日若出现“战备补给”事件还会再乘 90%。页面显示的就是服务端冻结后的有效价，旧页面的 offer 不会沿用到新版本。
4. 订单创建后保持角色在线。页面在可见且存在处理中订单时每 3 秒更新，进入终态后停止轮询。
5. `refunded` 表示已退款，可去资金流水核对；`cancelled` 表示订单取消；`partial` 表示只有部分物品被证明送达；`uncertain` 表示上游结果无法证明。遇到 `partial/uncertain` 不要再次购买同一补给，等待管理员按回执对账。

若提示 `OFFER_NOT_AVAILABLE`，刷新商城后按新版本重新选择；若提示全服库存售罄，不能通过重复点击绕过。

## 6. 出售白名单资源

1. 在常驻周世界中自由采集、战斗、生产、交易和囤积资源。系统不追踪资源的获得时间或来源。
2. 打开“资源兑换点地图”，查看今日被选中的动态开放区、风险等级、开放窗口、路线、范围、限时热点、**当前正在生效**的世界事件和实际收益倍率；尚未开始或已经结束的事件不会冒充当前倍率。可分别切换“开放区、关闭区、热点、风险、前往路线、我的位置”六个显示图层；切换只改变本机画面，右侧路线、窗口、风险、事件和收益文字仍来自服务端，不会被隐藏图层改写。默认每天只开放双点池中的 1 点，第 8–11 小时为热点；“资源繁荣”会把回收收益乘 115%，“战备补给”只影响商城价格。它们都是服务端经济证据，不代表额外掉落或击杀奖励。地图只显示你自己的位置，不显示其他玩家坐标；本人位置或本地底图不可用时仍保留权威文字和校准网格。候选点 2 仍待真实 Palworld 坐标验收。
3. 让角色进入**当前开放**的兑换区。进入关闭区只会显示关闭/下次开放提示，不会完成引导或允许出售；默认路线优先最近开放区，主动查看关闭区时会标明“预览/当前不可兑换”。服务端会间隔至少 2 秒连续采样位置，不能只靠浏览器声称“已到达”。若 5 秒位置轮询失败，旧地图可保留参考，但资格会立即降级为“上次位置不可确认”，不得据此扫描或出售。
4. 点击扫描。服务端读取 `Items`、`Food`、`DropSlot`，只对当前内容版本的白名单资源生成 30 秒源报价；扫描本身不扣物、不入账。
5. 报价中的资源默认全部勾选，每行显示本地图标、稀有度和用途；旧内容缺少展示字段时会标明由服务端 fallback 补全，展示差异不会改变 ItemID、数量、单价或结算权限。取消不想出售的资源，或在报价上限内调低出售数量；页面会实时重算已选种类、件数和战备券总额。至少选择一件资源后才能继续。内容发布器会拒绝让同一兑换区同时暴露超过 64 种可售 ItemID，确保默认全选仍可由一次原子结算处理。
6. 点击“复核所选资源”，再次核对 ItemID/名称、所选数量、单价、区域/热点/事件倍率和总额。确认提示必须明确“未选资源保留在背包”；内容版本、eventId、seed、窗口和倍率证据会原样冻结到所选子报价。
7. 在倒计时结束前点击“确认出售所选资源”。服务端会原子取消源报价并创建只能消费所选行的子报价，再结算该子报价。到期、重复/未知 ItemID、零/负数、超过报价数量、旧 revision 或内容指针变化都会无副作用拒绝。
8. 提交后不要移动相关物品或换一个按钮反复提交。页面会在当前会话中保存原选择键和结算键；断网、刷新或首次响应丢失后应按页面“重试”恢复，同一键只返回原子报价/原结算，不会再创建或再扣一次。
9. 只有 Native 完整证明逐项扣物、after 快照和聚合数量都与**所选报价**一致时，战备券才会唯一入账。Native 仍用扫描时完整背包快照防止并发移动，但扣物授权列表不包含未选资源。

热点或事件开始时，开始前的旧倍率报价到期；活动中的报价可在对应 `graceSeconds` 内且自身尚未过期时完成。所有点关闭时页面显示下一开放时间；内容版本改变会优先返回 `QUOTE_CONTENT_CHANGED`，即使仍处于 grace 也必须重新扫描。

兑换处理中的记录每 3 秒更新，并在终态后停止。`settled/已兑换` 才表示资源已扣且战备券已入账；`cancelled` 不会继续扣物；`uncertain/待人工核对` 时不要再次扫描或提交相关资源，联系管理员对账。

## 7. 查看周档结算

周世界切换后打开“周档结算”。页面会自动查询当前服务器最近一份**已经冻结**的周榜，不需要玩家知道或输入 `seasonId`；因此即使首页已经显示新一周，结算页仍会显示上一周的冻结结果。运营尚未冻结任何周榜时，页面只显示“尚未冻结”，不会拿实时数据推测临时名次或奖励。

结算页只返回当前登录账户，包含：

1. 资源价值榜与任务积分榜的个人成绩、名次，以及未参与、最低贡献不足、冻结时封禁或人工排除等明确原因；
2. 截止时间、冻结时间、最低贡献、结算宽限期和同分规则；
3. 按 ItemID/类别汇总的本人资源成绩，不包含其他玩家或全服排行榜；
4. 本周待过期或已过期的周战备券数量，以及过期任务是否已写入权威钱包账本；
5. 每一项永久商域币奖励决定，包括待发放、已取消、已发放和人工补发。只有显示“账本已确认/永久币已发放”才表示真正完成。

冻结时的历史名次不会因冻结后封禁而改写，但奖励可能显示“已取消”；页面会同时说明原因。若奖励显示“待发放”或周券显示“待过期/处理中”，等待运营结算，不要用重复购买、兑换或重新登录来尝试推动它。需要核对旧周时，受支持的本人只读接口仍可按 `seasonId` 查询，但网页默认使用安全的 `latest` 入口。

## 8. 查看消息中心并选择浏览器提醒

打开“消息中心”可查看只属于当前登录账户的持久通知。通知来自订单送达、资源兑换、周档结算和异常对账四类权威状态；刷新、重新登录或服务重启不会凭空新增同一条通知。使用“标为已读”或“全部标为已读”只改变阅读状态，不会再次购买、扣物、发奖或推动结算。

站内消息默认可用，浏览器系统提醒默认关闭。只有玩家主动点击“启用浏览器提醒”后，页面才会请求浏览器权限；拒绝权限、浏览器不支持或稍后关闭提醒都不影响站内消息。页面不会把通知正文、账户标识或 PlayerUID 存进浏览器偏好。

游戏内投递状态必须按页面原样理解：`已排队` 只表示等待游戏端处理，`已提交` 只表示命令已被游戏端接受，`blocked/当前不可用` 仍然只是站内消息，并不表示角色已经收到提示。出现 `partial/uncertain` 或“需要核对”时不要重复购买、扫描或提交同一经济动作；保留消息并联系管理员按订单/兑换编号对账。

## 9. 查看“我的服务器”

“我的服务器”只在管理员显式配置多服务器只读注册后提供有效聚合；默认关闭时会显示“尚未启用”，不是余额为 0。该页不会把当前 Cookie、验证码、SteamID、PlayerUID 或账户 ID 发给其他服务器，只使用不可逆联邦 subject 查询各节点摘要。

1. `可用` 只表示该节点成功返回并通过当前兼容矩阵校验；账户存在时才显示该服周档与双币余额。
2. `未找到本服账户` 表示该节点明确确认此联邦 subject 尚未创建账户，不等同于余额为 0。若你确信玩过该服，请联系管理员检查统一身份密钥版本，不能手工补余额。
3. `unavailable` 表示超时、节点下线、响应过大或安全校验失败；`incompatible` 表示游戏、PalDefender、UE4SS、Native 或矩阵版本不在允许组合。两种状态都不会伪装成 0，也不应靠重复刷新绕过。
4. “打开该服门户”只会打开管理员 allowlist 中的 HTTPS 地址，不会携带本服 session。切换后必须在目标门户重新登录并绑定该服当前周角色。
5. 当前不支持跨服转币、库存、队伍、交易或拍卖；每个服务器仍独占自己的 SQLite 写入面。

生产多服仍需真实第二/第三节点、TLS、独立节点密钥轮换和 `stable` 兼容组合验收。当前矩阵没有可供生产启用的 stable 组合，因此演示页与截图不能当作实服联调证据。

## 10. 核对资金流水并安全退出

资金流水记录购买扣款、退款、资源兑换和活动/任务奖励。完成购买或兑换后，应同时核对订单/兑换终态和对应账本分录；不要只凭游戏背包变化判断成功。

使用完毕后点击退出。会话过期会要求重新登录和绑定，不应通过刷新或重复提交绕过。断网、维护、队列拥堵、版本不匹配或角色离线时，页面会给出下一步；没有明确终态的经济操作应先恢复查询或联系管理员，不能创建新幂等请求“碰碰运气”。

## 11. 手机、键盘和屏幕阅读器操作

- 手机端底部“玩家功能”导航可切换商城、地图、订单、流水、兑换记录、团队协作、周档结算、消息中心和“我的服务器”，顶部始终保留“安全退出”。页面支持最窄 320 像素布局；若浏览器启用了超大字体或系统缩放，仍应先确认购买/出售弹窗完整显示再提交。
- 只用键盘时，首次按 `Tab` 会显示“跳到主要内容”；使用 `Tab`/`Shift+Tab` 移动、`Enter` 或空格触发按钮。购买或出售弹窗打开后焦点留在弹窗内，按 `Escape` 可在未提交时取消并返回原按钮。
- 切换功能后焦点会移到新页面标题；地图六个图层均为具名复选框，可用空格切换。登录、购买或兑换失败时焦点会移到错误提示，屏幕阅读器会立即播报错误和安全的下一步。关闭错误后焦点返回当前功能标题。
- 弹窗处理请求期间按钮会禁用，不能依赖 `Escape` 或重复点击中断已经提交的交易；应等待订单或兑换状态进入明确终态。

上述交互已用真实 Chromium 在 375×812 手机视口和纯键盘场景自动验收，并对登录页、玩家门户、地图图层、团队经济、周档结算页、消息中心和购买/出售弹窗执行 WCAG A/AA 的 Axe 严重/致命规则扫描。自动化基线不能替代具体手机、浏览器、缩放比例及实际读屏软件的上线抽查；管理员仍须执行 [玩家门户上线验收清单](04-验收清单.md)。
