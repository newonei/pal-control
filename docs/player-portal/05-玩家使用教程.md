# 玩家使用教程

本教程描述方案 A“周世界资源经济服”的当前玩家流程。服务器管理员必须先完成玩家门户、身份绑定、PalDefender、授权资源目录和 Economy Safety Gate 配置；若页面显示维护、版本不匹配或能力未就绪，请按提示等待管理员处理，不要反复提交。

> 当前 Native 扣物能力仍是 experimental，真实“扣物 -> 保存 -> 停服 -> 重启 -> 重登”验收前，Production 资源兑换会保持关闭。本文是受控开发服使用说明，不代表项目已经可以直接对公网开放。

## 1. 登录并绑定本周角色

1. 公网 Steam 模式会跳转到 Steam 官方 OpenID 页面。幻兽商域不会要求输入 Steam 或 Palworld 密码。
2. Steam 验证成功后只有短时待绑定身份，尚不能查看钱包或交易。
3. 保持本周 Palworld 角色在线，网页请求验证码；服务器通过本机受限通道向该角色私发 8 位一次性验证码。
4. 在网页提交验证码。服务端实时确认本周 `worldId + PlayerUID` 后才建立正式会话。
5. 可信好友服可能显式使用“仅游戏验证码”模式；页面会标出这是 fallback。不要把验证码、Cookie 或会话链接发给他人。

登录后首页按真实账户状态显示 5 步引导：登录绑定、购买补给、进入资源兑换点、确认整单出售、查看资金账本。步骤完成状态来自服务端订单、位置、报价、兑换和账本，不需要玩家手工打勾。

## 2. 领取新玩家活动

若当前周世界发布了新玩家活动，首页会显示版本、奖励与是否已领取。点击领取才会入账；同一活动版本只能领取一次，刷新或重放不会重复发奖。没有已发布活动时页面不会虚构默认奖励；开发兼容配置中的初始余额不代表正式服规则。

## 3. 查看任务

“本周可靠任务”显示日/周任务、当前进度、奖励、任务积分以及固定的规则/内容证据。内置内容候选为 3 个日任务和 3 个周任务，只累计服务端已经证明的经济事件，例如成功资源兑换、指定资源/价值结算、商城成功送达和指定货币消费。同一事件重放不会重复推进，同一奖励只入账一次。

击杀、采集、死亡、PvP 或进入热点目前不属于可靠任务事件；若页面没有这些任务，这是预期的安全边界。

## 4. 购买战备商品

1. 打开“战备商城”，按分类、标签或关键字筛选。推荐位、分类和标签由当前内容版本明确给出。
2. 区分两种数量：
   - “个人本周剩余”是你的周限购余量；
   - “全服剩余”只在商品配置了服务器总库存时显示，所有玩家共享。未配置全服库存时页面不会显示一个虚假库存数。
3. 核对商品、货币、当前价格和发放摘要后购买。目录每天按业务日生成版本化价格，旧页面的 offer 不会沿用到新版本。
4. 订单创建后保持角色在线。页面在可见且存在处理中订单时每 3 秒更新，进入终态后停止轮询。
5. `refunded` 表示已退款，可去资金流水核对；`cancelled` 表示订单取消；`partial` 表示只有部分物品被证明送达；`uncertain` 表示上游结果无法证明。遇到 `partial/uncertain` 不要再次购买同一补给，等待管理员按回执对账。

若提示 `OFFER_NOT_AVAILABLE`，刷新商城后按新版本重新选择；若提示全服库存售罄，不能通过重复点击绕过。

## 5. 出售白名单资源

1. 在常驻周世界中自由采集、战斗、生产、交易和囤积资源。系统不追踪资源的获得时间或来源。
2. 打开“资源兑换点地图”，查看当前开放状态、路线提示、范围和收益倍率。地图只显示你自己的位置，不显示其他玩家坐标；位置数据单独按地图刷新周期更新。
3. 让角色进入开放兑换区。服务端会间隔至少 2 秒连续采样位置，不能只靠浏览器声称“已到达”。
4. 点击扫描。服务端读取 `Items`、`Food`、`DropSlot`，只对当前内容版本的白名单资源生成 30 秒报价。
5. 逐项核对 ItemID/名称、数量、单价、区域倍率和总额。当前版本会把所有合格资源的全部数量放进同一报价：可以取消整单，但不能选择保留其中一部分。
6. 在页面倒计时结束前确认。到期后按钮自动禁用，服务端也会拒绝旧报价；兑换区关闭或内容版本改变时按提示重新扫描。
7. 确认后不要移动相关物品或重复提交。只有 Native 完整证明逐项扣物、after 快照和聚合数量都与报价一致时，战备券才会唯一入账。

兑换处理中的记录每 3 秒更新，并在终态后停止。`settled/已兑换` 才表示资源已扣且战备券已入账；`cancelled` 不会继续扣物；`uncertain/待人工核对` 时不要再次扫描或提交相关资源，联系管理员对账。

## 6. 核对资金流水并安全退出

资金流水记录购买扣款、退款、资源兑换和活动/任务奖励。完成购买或兑换后，应同时核对订单/兑换终态和对应账本分录；不要只凭游戏背包变化判断成功。

使用完毕后点击退出。会话过期会要求重新登录和绑定，不应通过刷新或重复提交绕过。断网、维护、队列拥堵、版本不匹配或角色离线时，页面会给出下一步；没有明确终态的经济操作应先恢复查询或联系管理员，不能创建新幂等请求“碰碰运气”。

部署和安全边界见 [玩家门户部署与安全资料](README.md)，玩法规则见 [ADR-0001](../architecture/decisions/0001-weekly-world-resource-economy.md)。
