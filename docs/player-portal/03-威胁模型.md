# 玩家门户威胁模型

## 1. 保护资产

- 玩家平台身份、登录会话、个人订单和钱包余额。
- 永久货币、周货币、商城订单、发货与撤离幂等记录。
- Palworld 管理员密码、PalDefender token、RCON、官方 REST 和 Native Bridge。
- 运营控制台、人工对账接口、换档接口和审计日志。
- Caddy TLS 私钥、ACME 账户状态和访问日志。

## 2. 信任区域

1. **不可信互联网**：浏览器输入、Host、Cookie、Authorization、查询参数和转发头都不可信。
2. **Caddy 公网边界**：终止 TLS、限制路径、提供静态文件并记录访问日志；不负责决定玩家能操作哪个账户。
3. **Player API**：验证平台登录、会话、CSRF、权限、资源归属、幂等和业务限制。
4. **本机游戏适配层**：Control API 才能调用官方 REST、PalDefender、RCON 和 Native Bridge。
5. **运营域**：管理员调账、对账、换档和配置接口；不得与玩家域共享路由或凭据。

## 3. 主要威胁与控制

| 威胁 | 影响 | 必须的控制 |
| --- | --- | --- |
| 绕过 Caddy 直连 `5180` | 获得完整 Control API 面 | 回环监听、无端口映射、防火墙与外部扫描 |
| 路径匹配过宽或 SPA 回退吞 API | 暴露管理/服务器接口 | 只代理 `/api/v1/player/*`；先拦截其他 `/api`；对编码路径做测试 |
| 伪造 `userId` 或修改他人资源 ID | 越权读取/消费 | 从服务端会话派生账户；对象级授权；不信任正文中的身份 |
| 会话盗用 | 钱包和商城被接管 | `Secure`、`HttpOnly`、合适的 `SameSite`、短会话、吊销与重新认证 |
| Cookie 写接口 CSRF | 未授权购买或操作 | CSRF token、严格 Origin 白名单与 SameSite Cookie；不能只依赖 CORS |
| XSS 或恶意构建资产 | 窃取会话、伪造操作 | CSP、无第三方任意脚本、依赖锁定、构建物审查、HttpOnly Cookie |
| 重放购买/领取请求 | 重复扣款或发货 | 强制幂等键、服务端请求哈希、唯一约束和终态重放 |
| 暴力登录、枚举、DoS | 账户攻击或服务不可用 | 验证码账户/IP限流；认证与玩家路径真实 IP 固定窗口；全局无排队并发上限；16 KiB 请求体上限；CDN/WAF 与监控 |
| Host/Forwarded 头注入 | 错误重定向、审计污染 | 站点域名匹配；上游 `Host=127.0.0.1`；只信任本机 Caddy |
| 访问日志泄密 | Cookie/token/个人信息泄露 | 不把令牌放查询串；保持凭据字段脱敏；限制 ACL、滚动和保留期 |
| 静态目录泄露 | `.env`、source map、源码或配置外泄 | 只发布干净 `dist`；拒绝点文件路径；无目录浏览；发布前递归清单和秘密扫描 |
| TLS 状态丢失或 ACME 滥用 | 证书中断、签发限额 | 持久化 Caddy data、最小权限、监控续期、不要频繁清空状态 |
| 人工对账接口被玩家调用 | 任意退款或入账 | 不代理 admin 路径；管理员认证/RBAC/双人审批仍在运营域完成 |

## 4. Caddy 安全头的意图

- HSTS 强制后续 HTTPS；模板不预设 `includeSubDomains` 或 `preload`，避免误伤尚未 HTTPS 化的其他子域。
- CSP 默认只允许同源脚本、样式和连接，禁止对象、frame、嵌入、内联脚本与内联样式。
- `X-Frame-Options: DENY` 与 `frame-ancestors 'none'` 防点击劫持。
- `nosniff`、严格 Referrer Policy、Permissions Policy 和同源资源策略减少浏览器攻击面。

任何新增第三方登录脚本、图片 CDN、WebSocket 域名或支付页面都必须先更新 CSP 设计和测试；不要为了“先能用”把 CSP 改成 `*` 或加入 `'unsafe-eval'`。

## 5. 残余风险与上线门槛

当前实现已经提供游戏内一次性验证码、HttpOnly 会话、严格 Origin、CSRF、用户/IP 限流、真实 IP 固定窗口、无排队并发上限、16 KiB 请求体上限、幂等和对象归属校验。进程内限流只用于保护单机业务，不能抵御多源流量；仍需根据正式运营规模补齐：

- 可选 Steam/平台登录的 state、nonce、回调域校验，以及与现有游戏内验证码身份的绑定策略；
- CDN/WAF 级 DDoS 防护、异常验证码告警与更细的风控、封禁策略；
- 跨 Control API 重启的会话持久化，以及管理员主动吊销全部玩家会话的能力；
- 审计日志、告警和隐私保留策略；
- 对游戏写入不确定结果的人工对账；
- 正式数据的备份恢复和数据库高可用。

这些能力没有通过安全测试前，Caddy 配置正确也只能算网络原型，不能称为正式玩家经济服。
