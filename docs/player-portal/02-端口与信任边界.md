# 端口与信任边界

## 1. 公网允许清单

| 端口 | 协议 | 用途 | 公网策略 |
| --- | --- | --- | --- |
| `80` | TCP | HTTP 到 HTTPS 重定向、ACME HTTP challenge | 仅转发到 Caddy |
| `443` | TCP | 玩家门户 HTTPS | 仅转发到 Caddy |
| `443` | UDP | HTTP/3 | 可选；不使用时关闭 |
| `8211` | UDP | Palworld 游戏流量 | 与门户独立，按游戏服需要映射 |
| `27015` | UDP | Palworld Query | 与门户独立，按服务器列表需要映射 |

门户上线不需要开放任何新的高位 TCP 管理端口。

## 2. 本机或内部专用清单

| 端口/通道 | 组件 | 必须保持的边界 |
| --- | --- | --- |
| `127.0.0.1:5180` | Control API | 仅 Caddy 与本机运营工具可访问；严禁路由器映射 |
| `127.0.0.1:5174` | 运营控制台开发站点 | 不属于玩家门户；严禁公网发布 |
| `8212/TCP` | Palworld 官方 REST | 防火墙阻断外部入站；严禁映射 |
| `127.0.0.1:17993` | PalDefender REST | 仅 Control API；严禁映射 |
| `25575/TCP` | RCON | 防火墙阻断外部入站；严禁映射 |
| `127.0.0.1:2019` | Caddy Admin API | 仅本机管理员；严禁映射 |
| Named Pipe | Native Bridge | 仅本机服务进程；没有 TCP 映射 |

## 3. HTTP 路径边界

| 请求 | Caddy 行为 |
| --- | --- |
| `/api/v1/player/*` | 原样反代到 `127.0.0.1:5180` |
| `/api` 或任何其他 `/api/*` | Caddy 直接返回 JSON `404` |
| 真实静态文件 | 从 `PLAYER_PORTAL_ROOT` 返回 |
| 其他非 API 路径 | 回退到 `index.html`，供 SPA 路由处理 |

`/api/v1/player`（末尾没有 `/`）不在代理白名单内，会返回 `404`。如未来确实需要该根路径，应先增加后端契约、威胁评审和验收用例，再精确增加匹配；不能把匹配器扩大为 `/api/v1/*`。

## 4. Host 与转发头

Caddy 站点地址只接受配置的玩家域名。到后端的 `Host` 被固定覆盖为 `127.0.0.1`，避免外部 Host 值穿透 Control API。Caddy会生成可信的 `X-Forwarded-For`、`X-Forwarded-Proto` 和 `X-Forwarded-Host`；Control API 只有在请求确实来自本机 Caddy 时才能信任这些头。

若前面再加 CDN 或云负载均衡，不能直接信任任意客户端提交的转发头。必须配置明确的可信代理地址、保留真实客户端 IP，并重新验证 Host、TLS、限流和日志；本模板默认假设 Caddy 是唯一公网入口。

## 5. 防火墙原则

- 入站默认拒绝，只为 Caddy 的 `80/443` 和明确需要的 Palworld UDP 端口放行。
- `5180` 即使当前监听回环，也继续在云安全组、路由器和 Windows 防火墙中禁止映射，形成纵深防御。
- RCON 与官方 REST 的 Block 规则不得因门户上线而删除。
- 从外部网络做端口扫描，验证 `5174/5180/8212/17993/25575/2019` 全部不可连接。
- 不使用“来源为任意地址”的临时远程管理放行规则；维护使用受控 VPN 或本机登录。
