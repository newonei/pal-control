# 玩家门户部署与安全资料

本目录定义 `apps/player-web/dist` 的公网边界。它只允许玩家门户静态页面和 `/api/v1/player/*`，不把 Control API、运营控制台、Palworld 官方 REST、PalDefender、RCON 或 Native Bridge 暴露给互联网。

当前玩家功能包括本人钱包、带本地 SVG 图标/稀有度/用途的版本化商城、订单、流水、新玩家活动、可靠日/周任务、5 步新手引导、选择性资源兑换、六图层兑换点地图、同服周世界团队协作、本人周档结算、本人消息中心，以及默认关闭的多服务器只读注册页。周档结算默认读取最近冻结周榜，即使当前周已切换也不会要求玩家提供 UUID；名次来自不可变快照，周券过期和永久币发放状态逐项与 season job 及权威钱包账本核对。消息中心从权威订单、兑换、周档与异常对账状态生成持久 feed；浏览器系统提醒只在玩家主动点击后授权，拒绝/不支持时仍显示站内消息。处理中订单/兑换在页面可见时每 3 秒有界轮询并在终态停止；消息轮询还要求页面可见且存在未读/投递/经济活动或玩家正在消息页。报价显示倒计时并在到期后禁用。兼容路径
`GET /api/v1/player/me/extraction-zones` 只从 HttpOnly 会话确定玩家，返回本人的位置与公开的
资源兑换点配置；不会返回其他玩家位置。页面每 5 秒刷新，位置不可用时明确显示未知而不会推测。开放区、关闭区、热点、风险、路线和本人位置图层均为显示开关，不会修改服务端事实或交易资格。

玩家端还提供跳到主要内容、清晰的焦点轮廓、功能切换后的标题焦点、错误提示播报和弹窗焦点圈定/返回。真实 Chromium E2E 覆盖纯键盘购买、错误恢复、375×812 手机导航、选择性出售、地图图层、团队协作、三节点只读注册、周档结算、消息已读/浏览器显式授权与弹窗，以及相应页面的 Axe WCAG A/AA 严重和致命规则扫描；命令为 `npm run test:player:e2e`。这属于提交级自动化基线，不能替代真实域名、手机和读屏软件的上线抽查。

## 不能混淆的安全结论

1. Caddy 的路径白名单只是网络边界，不是玩家身份认证。`/api/v1/player/*` 仍必须由 Control API 校验登录会话、权限、CSRF、幂等键和资源归属。
2. 当前模板要求 Caddy 与 Control API 在同一台 Windows 主机上，后端只监听 `127.0.0.1:5180`。若拆成两台主机，不能简单把 `5180` 开到公网，必须重新设计受认证的私网或 mTLS 链路。
3. `apps/player-web/dist/index.html` 不存在、Control API 不是回环监听、玩家 API 尚未完成认证，三者任一不满足都不得上线。
4. 运营控制台 `5174` 与玩家门户是两个安全域。不要把运营页面复制进玩家静态目录，也不要给玩家站点增加 `/api/v1/extraction/admin/*`、`/api/v1/servers/*` 等代理规则。
